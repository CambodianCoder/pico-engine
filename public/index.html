Pico Bootstrap

Operations log



Manual next steps



Check the operations log, above.
It shows the creation of the Pico and its channel,
and the registering of the two
rulesets which are needed by every Pico.



Enable and install rulesets



Visit the Engine Rulesets
page for each ruleset.
While there, click on the enable button
and then on the install button for each.



Add rulesets to the Pico



Visit the My Picos
page and in the Rulesets tab for the Owner Pico,
add the two rulesets.



Record Owner Pico ids



The final bootstrap step is to use
the Owner Pico ruleset to record
its identifiers.
To do this, visit its About tab
and click on the link
"send event pico/root_created"
to inform it that it is
the root of your Pico hierarchy.



Done



The bootstrap is now complete.
You may close this tab and use
the other two tabs to create
your Pico hierarchy and
add your own rulesets.



Reset database



If you wish to start over,
it is safe to stop your
HTTP server (use Ctrl-C)
and remove the database.
Use rm -r db
to clear the database.
Then restart the HTTP server
(use npm start)
which will create an empty
database.
Refresh this Bootstrap page
and complete the bootstrap
process each time you
clear the database.
